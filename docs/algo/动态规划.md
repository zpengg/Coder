
# 动态规划笔记
主要是为了解决*重叠子问题*

## 步骤
 - 明确状态
 - 明确选择
 - 明确base case 
 - 写出状态转移方程
 - 状态压缩优化

## 难点
状态数组：备忘录，可多维度
状态转移方程

### 暴力分析
画出递归树，便于分析。
分析是否有子问题重叠，时间复杂度太大

### 自顶向下
> [!TIP]
> 状态转移方程通常即已经是递归逻辑

需要补充的是，记录之前递归计算过的结果，对递归树进行减枝
自顶向下解决问题

### 自底向上
动态规划 则是 自底向上
> [!TIP]
> 先确定好 base case ,通常是边界值

## 多位度状态数组的遍历方向
